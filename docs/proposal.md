## 📄 p1. 기획배경 및 필요성

### 1. 서비스 기획 배경

2024~25년 외국인 관광객 대상 “바가지” 민원은 해운대·명동·부산을 중심으로 지속 이슈화되었습니다. 한국관광공사가 발표한 『외국인 친화적 관광 환경 조성 계획』의 핵심은 **정보 비대칭 해소**와 **지역 분산 관광 활성화**이지만, 현장에서 외국인 관광객이 곧바로 쓸 수 있는 **공공데이터 기반의 실시간 가격 참조 도구**는 부재합니다.

### 2. 서비스 필요성

- **정보 비대칭** — 메뉴판·요금이 외국어로 표기되지 않은 곳이 여전히 다수이며, 정상 가격 참조 출처가 SNS 후기로 분산되어 있습니다.
- **서울 집중** — 서울 외 지역은 관광 정보가 부족해, 정책상 강조되는 지방 분산 관광의 진입 장벽이 높습니다.
- **언어 장벽** — 관광정보의 4개 언어(국문·영·일·중) 동시 제공 서비스가 제한적입니다.

### 3. 목표 지표

- 접수 시점까지 **정상가 판정 + 테마 여행 + 공공 POI 탐색**을 한 서비스에서 4개 언어로 제공
- 경기도 부천시를 **지역 특화 베드**로 삼아, 공공데이터 LIVE 연동을 실증

## 📄 p2. 서비스 개요

### 한 줄 소개

> **외국인 관광객을 위한 부천 중심 한국 여행 안전망**
> — 공공데이터 기반 정상가 판정 + 31개 테마 여행 + 관광 POI 탐색

### 핵심 기능 5개

| #   | 기능               | 설명                                          | 데이터 소스                   |
| --- | ------------------ | --------------------------------------------- | ----------------------------- |
| 1   | **바가지 판정**    | 지불 금액과 부천시 개인서비스요금을 LIVE 비교 | 부천시 공공데이터(data.go.kr) |
| 2   | **부천 테마 여행** | 31개 코스 × 13 스타일 × 5 기간                | 자체 큐레이션 + TourAPI       |
| 3   | **관광 지도**      | 부천·인근 495+ POI 4개 언어                   | KTO TourAPI                   |
| 4   | **사용자 참여**    | 의견·좋아요로 관광 상품 개발 피드백           | 자체 Firestore                |
| 5   | **오프라인 활용**  | PDF 인쇄·Web Share                            | Web API                       |

### 차별점

- **공공데이터 LIVE 연동** — mock 데이터가 아닌 `source: 'live' \| 'mock'` 뱃지로 검증
- **4개 언어 동시 지원** — TourAPI `KorService2 / EngService2 / JpnService2 / ChsService2` 병렬
- **지역 특화 가점(+2)** — 경기도 부천시(서울 제외) 집중으로 RTO 특별상 제외 대신 1개 지역 특화 가점 확보

### 부천 특화 근거

- **K-콘텐츠 자산** — BIFAN(부천국제판타스틱영화제), 한국만화박물관, 부천아트센터 등
- **접근성** — 인천공항 30분 · 서울 30~40분 (지하철 1·7호선 + 공항철도 연계)
- **재래시장 인프라** — 심곡본동·자유시장·원미시장 등 10곳 이상의 로컬 상권

## 📄 p3. 데이터 활용 방안

### 활용 예정 한국관광공사 OpenAPI

| 엔드포인트       | 용도                                       | 언어            |
| ---------------- | ------------------------------------------ | --------------- |
| `areaBasedList2` | 지역·시군구 기반 POI 일괄 수집 (495건)     | Kor/Eng/Jpn/Chs |
| `detailCommon2`  | 선택 POI 상세 텍스트                       | Kor/Eng/Jpn/Chs |
| `searchKeyword2` | 키워드 검색 (Cloud Function `tour-api.ts`) | Kor/Eng/Jpn/Chs |
| `areaCode2`      | 시·도 코드 매핑                            | Kor             |
| `ldongCode2`     | 법정동 코드 매핑 (지역 분류)               | Kor             |

**→ 5개 엔드포인트 × 4개 언어 서비스 = 20개 조합 실증.**

### 데이터 플로우

```
┌─────────────────────────────────────┐     ┌────────────────────────────┐
│   한국관광공사 TourAPI (data.go.kr)  │     │   부천시 개인서비스요금      │
│   KorService2 · EngService2        │     │   (경기도 공공데이터)        │
│   JpnService2 · ChsService2        │     │                            │
│   ─ areaBasedList2                 │     │   ─ 품목별 min/max/samples │
│   ─ detailCommon2                  │     │   ─ LIVE 조회               │
│   ─ searchKeyword2                 │     │                            │
│   ─ areaCode2 / ldongCode2         │     │                            │
└──────────────┬──────────────────────┘     └──────────────┬─────────────┘
               │                                             │
               ▼                                             ▼
  ┌────────────────────────────┐           ┌─────────────────────────────┐
  │  POI 인덱스 495건           │           │  바가지 판정 엔진            │
  │  ─ 부천 코어 115건          │           │  ─ 지불 금액 ↔ 정상가 구간   │
  │  ─ 인근 380건              │           │  ─ verdict: fair/careful/   │
  │  ─ 4개 언어 번역 캐시        │           │    bagaji                   │
  └────────────┬───────────────┘           └─────────────┬──────────────┘
               │                                            │
               └────────────────┬───────────────────────────┘
                                ▼
                    ┌──────────────────────┐
                    │  NWK 앱               │
                    │  ─ 31개 부천 코스     │
                    │  ─ 관광 지도          │
                    │  ─ 판정 LIVE 뱃지    │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  사용자 참여 루프     │
                    │  ─ 의견·좋아요        │
                    │  → 관광 상품 개발     │
                    │    피드백             │
                    └──────────────────────┘
```

### 사용자 참여 데이터 (Firestore)

- **의견(notes)** — 각 코스별로 수집, 관리자 모더레이션 후 공개
- **좋아요(likes)** — 코스 선호도 집계, 향후 추천 알고리즘 입력
- **수집 기간** — 접수 이후 지속, 분기별 코스 리뷰 기준

### 대표 스크린샷 (실제 출력 시 가로 2×3 배치)

1. 메인 대시보드 — 정상가 + 부천 코스 프리뷰
2. 바가지 판정 결과 — LIVE 뱃지 노출
3. 부천 테마 여행 리스트 — 스타일·기간 필터
4. 코스 상세 — 타임라인 + 의견 섹션
5. 관광 지도 — 번호 핀 + 바텀시트 4개 언어
6. 4개 언어 전환 비교 화면

### 기술 스택

- **Frontend** — React 19 · TypeScript · Vite · TailwindCSS 4
- **Map** — Kakao Maps JS SDK (해외 접근성 대응, VWorld → Kakao 전환)
- **Backend** — Firebase Auth · Firestore · Functions · Hosting
- **Data** — KTO TourAPI · 부천시 개인서비스요금 공공데이터
- **Live** — https://nwk-app-ba6f8.web.app

### 완성도 수치

- **코스** 31개 · **POI** 495+ · **가격 카탈로그** 30+ 품목
- **지원 언어** 4개 (ko/en/ja/zh) 전 경로 커버
- **PWA** — 홈 화면 설치 + 오프라인 최근 조회
- **공모전 접수 시점 기준 라이브 서비스 운영 중**

## 📄 p5. 발전 방향

### 3단계 로드맵

#### 🟢 3개월 내 — 관리자 CMS 정식 오픈

- `/admin/courses` 편집 UI (이미 MVP 구현 완료)
- `/admin/notes` 의견 모더레이션 (이미 MVP 구현 완료)
- Firestore 실시간 동기화 — 정적 JSON과 이중 운영
- 관광 상품 매니저 계정(Firebase Auth custom claim) 배포

#### 🟡 6개월 내 — 광역 15개 시도 확장

- 광역 시·도별 개인서비스요금 공공데이터 순차 연동
- 시·도별 관광 상품 매니저 인터페이스
- RTO 특별상 대상 지역(부산·대전·광주·세종·충남·경북·강원·제주)을 우선 온보딩

#### 🔵 12개월 내 — AI 코스 추천

- 누적된 좋아요·의견을 바탕으로 개인화 추천
- 4개 언어 Q&A 챗봇 (Claude/GPT API 운영비 확보 후)
- 관광 상품 개발 데이터 기여 — 부천시 관광 정책 피드백 루프

### 기대 효과

- **체류시간 연장** — 단일 POI 중심 여행 → 코스 단위 다점 여행으로 1~3박 유도
- **지방 분산 관광** — 부천 모델의 타 지역 복제 가능성 검증
- **관광 상품 개발 데이터 자산화** — 의견·좋아요 → 관광정책 피드백
