# NWK — 제안서 스크린샷 촬영 체크리스트

> 5/6 접수 제안서 PDF에 삽입할 대표 스크린샷 6장. 각 샷의 구도·해상도·주석 포인트를 명시.

## 📸 촬영 환경

- **해상도**: 1920×1080 desktop 기본, 주요 장면은 iPhone 14 Pro viewport (390×844) 모바일 버전도 1장씩 준비
- **브라우저**: Chrome 최신, DevTools 디바이스 에뮬레이션 또는 실제 모바일
- **도메인**: `https://nwkorea.com` (DNS 연결 후) 또는 백업으로 `https://nwk-app-ba6f8.web.app`
- **언어**: 1·2·3·4번은 한국어 UI, 5번은 영어 UI, 6번은 일본어 UI (4언어 지원 증빙)
- **저장**: PNG, 손실 없이, 파일명 `nwk-01-hero.png` 형식
- **권장 크기**: 가로 1600px 이상, 300dpi 환산 → 제안서 지면 가로 8cm 기준 2367px → 1600~1920 OK

## 🎯 6장 스펙

### 1. 메인 대시보드 + 바가지 히어로 피드 (`/`)

- **언어**: 한국어
- **구도**: 상단 헤더(로고+탭+언어) + 히어로(바가지 판정 카드 3장 데스크톱, 6장 모바일 스냅) 포함
- **체크 포인트**:
  - [ ] "바가지? 걱정 말고 여행해요" 헤드라인 명확히 보임
  - [ ] 히어로 카드에 **실제 publicChecks 데이터 3~6건** 노출 (시딩 필수)
  - [ ] 이메일 마스킹 (`m*******@nate.com` 포맷) 확인
  - [ ] 카드 배경 카테고리별 다른 이미지 (food/drink/transit/...) 시각 구분

### 2. 바가지 판정 결과 — LIVE 뱃지 (`/check` step 4)

- **언어**: 한국어
- **시나리오**: 카테고리 → 음료 → 생수(500ml) → 5000원 입력 → 판정하기
- **체크 포인트**:
  - [ ] 큰 빨간 "바가지 가능성" 배지
  - [ ] "+280%" 수치 명확
  - [ ] 하단 **"실데이터 출처"** 카드에 `경기도 부천시 개인서비스요금 현황 (2025-05)` 링크 보임
  - [ ] 또는 **"공식 근거"** 카드로 `한국소비자원 참가격 외식비 통계` 링크 중 1개 포함

### 3. 부천 테마 여행 리스트 (`/courses`)

- **언어**: 한국어
- **구도**: 상단 스타일·기간 필터 + 코스 카드 그리드
- **체크 포인트**:
  - [ ] 13개 스타일 태그 버튼 한눈에
  - [ ] 코스 카드 최소 6~9장 (2x3 또는 3x3) 보이는 뷰
  - [ ] 각 카드에 기간 배지(half-day/day/1n2d 등) + 스타일 태그 2~3개
  - [ ] 공모전 지역 특화 근거 — "부천" 지역명 최소 1장에서 강조

### 4. 코스 상세 + 의견 섹션 (`/courses/:id`)

- **언어**: 한국어
- **시나리오**: 대표 코스 1개 (예: "한국만화박물관 → 상동호수공원 → 부천 중앙공원")
- **체크 포인트**:
  - [ ] 히어로 이미지 + 코스 제목
  - [ ] 번호 순서대로 스톱 타임라인 (번호 핀 아이콘)
  - [ ] 하단 사용자 의견(notes) 최소 1~2건 (시딩 필요 시 admin 계정으로 작성)
  - [ ] "공유" "좋아요" 액션 버튼
  - [ ] 모바일 뷰도 1장 별도 촬영 (390×844)

### 5. 관광 지도 + 4언어 툴팁 (`/map`)

- **언어**: **영어**
- **시나리오**: 코스 선택 후 번호 핀 클릭 → 한국만화박물관 또는 부천 중앙공원 툴팁
- **체크 포인트**:
  - [ ] 툴팁에 **영어 타이틀** ("Korea Manhwa Museum")
  - [ ] **영어 주소** ("1 Giljuro-ro, Wonmi-gu, Bucheon-si, Gyeonggi-do")
  - [ ] **영어 설명** 2~3줄
  - [ ] 좌측 사이드바에 코스 스톱 리스트 영문 타이틀 (일관성)
  - [ ] "공공 데이터 4언어 완전 활용" 제목 캡션 옵션

### 6. 4개 언어 전환 비교 (합성)

- **구성**: 같은 화면(예: 홈 또는 코스 리스트)을 4개 언어로 캡처해 **2x2 합성**
- **체크 포인트**:
  - [ ] ko / en / ja / zh 4컷 동일 뷰포트·동일 스크롤 위치
  - [ ] 각 컷 우상단에 언어 태그 (KO/EN/JA/ZH)
  - [ ] 같은 코스 이름이 "한국만화박물관 / Korea Manhwa Museum / 韓国漫画博物館 / 韩国漫画博物馆" 처럼 즉시 다국어 전환되는 증거
  - [ ] 주석으로 "All 4 languages powered by TourAPI + manual curation" 한 줄

## 🛠 촬영 전 준비

1. **라이브 데이터 시딩** — 히어로가 비면 임팩트 약함. 관리자 계정으로 3~5건 판정을 미리 `publicChecks`에 제출.
2. **로그인 상태** — 판정 공유·의견 작성 경로는 로그인 상태여야 전체 기능 노출.
3. **다크모드 해제** — 제안서는 라이트모드 통일 (심사위원이 인쇄 시 잉크 소모 고려).
4. **캐시 초기화** — 새로고침 후 캡처 (오래된 UI 버전 방지).
5. **브라우저 UI 제거** — DevTools 디바이스 에뮬레이션으로 깔끔한 뷰포트, 또는 Chrome 전체화면.
